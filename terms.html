Academic Information

End-Semester Project, Spring 2023

1. Course Details

This Restaurant Billing System was developed as part of the end-semester project for the User Interface Design (UID) course in the Computer Science department. The project was designed to demonstrate students' ability to apply UI/UX principles in a practical application.

The project incorporates various aspects of user interface design including responsive layouts, color theory, information architecture, and user-centered design principles.

2. Project Requirements

The project had to fulfill the following academic requirements:

2.1 User Interface Design

  • Implement a responsive design that works on multiple screen sizes
  • Create a consistent visual language (color scheme, typography, icons)
  • Design intuitive navigation and information architecture
  • Incorporate accessibility features (e.g., dark mode, clear contrast)

2.2 Functionality Requirements

  • Authentication system with login/logout functionality
  • Menu management with categorization
  • Billing system with item selection, quantity adjustment, and total calculation
  • Order tracking and management
  • Reporting and analytics visualization
  • Data persistence (implemented using session storage)

2.3 Technical Implementation

  • Clean, well-structured HTML5 markup
  • CSS3 styling with modern features (flexbox, grid, variables)
  • Vanilla JavaScript for interactive functionality
  • No reliance on external libraries or frameworks
  • Cross-browser compatibility

3. Learning Outcomes

Through this project, students were expected to demonstrate the following learning outcomes:

3.1 Design Skills

  • Apply visual design principles to create an aesthetically pleasing interface
  • Design intuitive user flows and navigation patterns
  • Create responsive layouts that adapt to different screen sizes
  • Implement accessibility best practices

3.2 Technical Skills

  • Write clean, semantic HTML structure
  • Implement modern CSS techniques for styling and layout
  • Create interactive elements using JavaScript
  • Implement client-side data management and persistence

3.3 Project Management

  • Work effectively in a team environment
  • Manage project timelines and deliverables
  • Document design decisions and technical implementations
  • Present and defend design choices to stakeholders

4. Evaluation Criteria

The project was evaluated based on the following criteria:

4.1 User Interface (40%)

  • Visual design and aesthetics
  • Layout and responsive behavior
  • Navigation and information architecture
  • Consistency and adherence to design principles

4.2 Functionality (30%)

  • Completeness of required features
  • Accuracy of calculations and data handling
  • Performance and responsiveness
  • Error handling and user feedback

4.3 Technical Implementation (20%)

  • Code quality and organization
  • Proper use of HTML, CSS, and JavaScript
  • Browser compatibility
  • Accessibility compliance

4.4 Documentation and Presentation (10%)

  • Project documentation quality
  • Presentation clarity and professionalism
  • Ability to explain design decisions
  • Responsiveness to questions and feedback

5. Academic Timeline

5.1 Project Milestones

  • Week 1-2: Project proposal and requirements gathering
  • Week 3-4: Wireframing and visual design
  • Week 5-8: Core implementation (HTML, CSS, JavaScript)
  • Week 9-10: Testing, refinement, and bug fixes
  • Week 11: Documentation completion
  • Week 12: Final presentation and submission

5.2 Deliverables

  • Project proposal document
  • Wireframes and mockups
  • Working prototype
  • Final implementation
  • Technical documentation
  • User guide
  • Presentation slides
